## 1. Тема и концепция

**Сеть моих коттеджей «Гордеют»**

* Главная цель — разработать веб-приложение, позволяющее пользователям изучить весь каталог коттеджей и выбрать наиболее подходящий вариант
* Веб-сайт предоставляет возможность забронировать домик на определённую дату, при этом система контролирует, соответствует ли клиент условиям, установленным арендодателем
* При возникновении вопросов на сайте предусмотрен удобный чат, в котором пользователь может задать любой интересующий его вопрос и получить необходимую помощь.

**Целевая аудитория:**
* незарегистрированные пользователи, просматривающие каталог
* зарегистрированные пользователи, бронирующие коттеджи
* арендодатель, размещающий свои коттеджи

**Ключевой функционал:**
* Просмотр каталога коттеджей с подробным описанием;
* Выборка подходящих вариантов коттеджей по дате, цене, кол-ву гостей;
* Бронирование коттеджа;
* Управление текущими заявками;
* Чат пользователя с арендодателем.

## 3. Технологический стек

**Frontend**
* Фреймворк: React — популярная и проверенная библиотека для динамических интерфейсов SPA.
* Роутер: React Router — удобная маршрутизация с поддержкой приватных и динамических страниц.
* State-менеджмент: Redux Toolkit — управляет состоянием пользователя, бронирований и чата централизованно.

**Backend**
* Язык: Node.js — один язык с фронтендом, быстро обрабатывает запросы, идеально для real-time чата.
* Фреймворк: Express.js — лёгкий и гибкий для создания REST API.
* База данных: PostgreSQL — надёжная реляционная СУБД с поддержкой сложных запросов и транзакций.

**Инфраструктура**
* Хостинг Frontend: Vercel — быстрый деплой SPA с CDN и HTTPS.
* Хостинг Backend: Render — простой деплой Node.js и масштабирование API.
* Хостинг БД: Supabase — облачный PostgreSQL с real-time API и удобным управлением.

## 4. Визуальное проектирование интерфейса

**Главная (/)**
* Тип доступа: публичный
* Основные блоки:
    1. Шапка: логотип, меню (Главная, Каталог, Войти), аватар (если авторизован)
    2. Блок приветствия: заголовок, кнопка "Перейти в каталог"
    3. Секция «Популярное»: 6 карточек коттеджей (данные из GET /api/items?popular=true)
    4. Футер: контакты, ссылки на соцсети
* Динамический контент: блок 3 (карточки коттеджей подгружаются с API)

**Аутентификация (/auth/*)**
* Тип доступа: публичный
* Основные блоки:
    1. Форма входа: email, пароль, кнопка "Войти"
    2. Форма регистрации: имя, email, пароль, подтверждение пароля, кнопка "Зарегистрироваться"
    3. Восстановление пароля: email, кнопка "Отправить инструкцию"
* Динамический контент: ошибки формы, уведомления об успешной регистрации или восстановлении

**Каталог (/catalog)**
* Тип доступа: публичный
* Основные блоки:
    1. Фильтры: цена, количество мест, удобства, дата
    2. Список коттеджей: карточки (данные из GET /api/items)
    3. Пагинация: переключение между страницами каталога
* Динамический контент: карточки коттеджей и фильтры подгружаются с API

**Детальная страница (/catalog/:id)**
* Тип доступа: публичный
* Основные блоки:
    1. Галерея изображений коттеджа
    2. Информация: название, описание, характеристики, цена
    3. Кнопка "Забронировать"
    4. Секция «Похожие коттеджи»
* Динамический контент: данные о коттедже и похожие объекты подгружаются с API

**Личный кабинет (/profile)**
* Тип доступа: приватный
* Основные блоки:
    1. Личная информация: имя, email, телефон
    2. История бронирований: список с датами, статусом и деталями брони
    3. Настройки аккаунта: смена пароля, уведомления
* Динамический контент: данные пользователя и бронирования подгружаются с API

**Бронирование (/booking)**
* Тип доступа: приватный
* Основные блоки:
    1. Форма бронирования: выбор даты, количество гостей, дополнительные услуги
    2. Подтверждение брони: стоимость, условия и детали брони
    3. Кнопка "Оплатить и подтвердить"
* Динамический контент: доступные даты и стоимость подгружаются с API

**Чат (/chat)**
* Тип доступа: приватный
* Основные блоки:
    1. Список диалогов
    2. Окно чата: сообщения, поле ввода текста, кнопка отправки
    3. Уведомления: о новых сообщениях
* Динамический контент: сообщения чата обновляются в реальном времени через API

**Админ-панель (/admin)**
* Тип доступа: приватный
* Основные блоки:
    1. Управление пользователями: список, поиск, блокировка
    2. Модерация коттеджей: добавление, редактирование, удаление объявлений
    3. Статистика: бронирования, активность пользователей
* Динамический контент: данные пользователей и коттеджей загружаются с API

## 5. Схема данных и API

**1. Модели данных**

| Модель      | Поля          | Тип              | Описание                                    |
| ----------- | ------------- | ---------------- | ------------------------------------------- |
| **User**    | id            | integer, PK      | Уникальный идентификатор пользователя       |
|             | name          | string           | Имя пользователя                            |
|             | email         | string, unique   | Email для входа                             |
|             | password_hash | string           | Хеш пароля                                  |
|             | role          | string           | Роль (`user` / `admin`)                     |
|             | phone         | string, optional | Телефон                                     |
|             | created_at    | datetime         | Дата регистрации                            |
|             | updated_at    | datetime         | Дата последнего обновления                  |
| **Cottage** | id            | integer, PK      | Уникальный идентификатор коттеджа           |
|             | title         | string           | Название коттеджа                           |
|             | description   | text             | Подробное описание                          |
|             | price         | decimal          | Стоимость за ночь                           |
|             | location      | string           | Город / адрес                               |
|             | guests        | integer          | Максимальное количество гостей              |
|             | amenities     | array/string     | Удобства (Wi-Fi, бассейн и т.д.)            |
|             | images        | array/string     | Ссылки на фотографии                        |
|             | is_popular    | boolean          | Популярный коттедж                          |
|             | created_at    | datetime         | Дата добавления                             |
|             | updated_at    | datetime         | Дата последнего обновления                  |
| **Booking** | id            | integer, PK      | Уникальный идентификатор бронирования       |
|             | user_id       | FK → User.id     | Пользователь, который бронирует             |
|             | cottage_id    | FK → Cottage.id  | Коттедж, который бронируется                |
|             | start_date    | date             | Дата начала бронирования                    |
|             | end_date      | date             | Дата окончания бронирования                 |
|             | status        | string           | Статус (`pending`, `confirmed`, `canceled`) |
|             | total_price   | decimal          | Итоговая стоимость бронирования             |
|             | created_at    | datetime         | Дата создания записи                        |
|             | updated_at    | datetime         | Дата последнего обновления                  |

**2. Ключевые эндпоинты API**

| Метод | URL                  | Описание                               | Тело запроса                | Ответ                              |
| ----- | -------------------- | -------------------------------------- | --------------------------- | ---------------------------------- |
| POST  | `/api/auth/register` | Регистрация нового пользователя        | `{ name, email, password }` | `{ id, name, email, token }`       |
| POST  | `/api/auth/login`    | Вход пользователя                      | `{ email, password }`       | `{ id, name, email, token }`       |
| GET   | `/api/users/me`      | Получение данных текущего пользователя | `{ email, id }`             | `{ id, name, email, role, phone }` |

| Метод | URL                     | Описание                                      | Тело запроса                                | Ответ                                                                                |
| ----- | ----------------------- | --------------------------------------------- | ------------------------------------------- | ------------------------------------------------------------------------------------ |
| GET   | `/api/cottages`         | Получение списка всех коттеджей с фильтрацией | `?price_min=&price_max=&guests=&amenities=` | `[ { id, title, price, location, guests, images, is_popular } ]`                     |
| GET   | `/api/cottages/:id`     | Получение деталей одного коттеджа             | —                                           | `{ id, title, description, price, location, guests, amenities, images, is_popular }` |
| GET   | `/api/cottages/popular` | Получение популярных коттеджей                | —                                           | `[ { id, title, price, images } ]`                                                   |

| Метод | URL                | Описание                                            | Тело запроса                                   | Ответ                                                                    |
| ----- | ------------------ | --------------------------------------------------- | ---------------------------------------------- | ------------------------------------------------------------------------ |
| POST  | `/api/bookings`    | Создание бронирования                               | `{ cottage_id, start_date, end_date, guests }` | `{ id, user_id, cottage_id, start_date, end_date, total_price, status }` |
| GET   | `/api/bookings/me` | Получение списка бронирований текущего пользователя | `{ "status": "active", "limit": 10 }`          | `[ { id, cottage_id, start_date, end_date, total_price, status } ]`      |
